The rentrez package provides an interface to NCBI's Entrez utilities.

Description

NCBI provides the EUtilities web service for software to query the Entrez databases directly, rather than through the web interface. For more information see here.

Details

This package provides an interface to construct query URLs and parse query results.

Important functions:

  • einfo: Provides field names, term counts, last update, and available updates for each database.

  • esearch: Searches and retrieves primary UIDs (for use with epost, esummary, elink, or efetch)

  • epost: Uploads primary UIDs to the Entrez history server for subsequent use with e.g. esummary, elink, or efetch)

  • esummary: Retrieves document summaries (DocSums) from a list of primary UIDs.

  • elink: Returns a list of UIDs (and relevancy scores) from a target database that are related to a list of UIDs in the same database or in another Entrez database.

  • efetch: Retrieves data records in a specified format corresponding to a list of UIDs.

Examples

############################################################################
# combine esearch with esummary/efetch

# Download PubMed records that are indexed in MeSH for both 'Clamydia' and 
# 'genome' and were published in 2011.
query <- "Chlamydia[mesh] and genome[mesh] and 2011[pdat]"

# check how many records fit the bill and download the PMIDs
n <- ecount(query, db="pubmed")


ESearch query using the ‘pubmed’ database.

pmids <- esearch(query, db="pubmed", retmax=n)

# Have a quick look at the titles
titles <- docsum(esummary(pmids))$title

# combine epost with esummary/efetch

# Download protein records corresponding to a list of GI numbers.
id_list <- c(194680922,50978626,28558982,9507199,6678417)

# post the GI numbers to the Entrez history server
p <- epost(id_list, "protein")

# retrieve docsums with esummary
docsums <- docsum(esummary(p))
